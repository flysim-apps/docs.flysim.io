# Settings
{: .no_toc }

## Table of contents
{: .no_toc .text-delta }

1. TOC
{:toc}

---

The **Settings** screen lets you configure FlyAround's behavior, audio, streaming overlay, and AI assistant options.

---

## Opening Settings

Click **Settings** in the main navigation menu.

---

## System Tab

### General Settings
- Global application configuration and behavior preferences

### Local Settings

| Setting | Description |
|---------|-------------|
| **Parking Mode** | Toggle parking behavior on/off |
| **Ground Handling** | **GSX** (GSX ground services add-on) or **SIM** (default simulator ground handling) |
| **Announcements** | Toggle individual cabin/ground announcement types on or off |
| **Volume Controls** | Adjust audio levels for different announcement categories |

{: .note }
**Ground Handling — GSX vs SIM:** If you use the GSX add-on for pushback and boarding, set this to "GSX" so FlyAround correctly tracks those ground phases. Use "SIM" for default MSFS ground services.

---

## AI Assistant Tab

*(Visible only if the AI feature is enabled on your account)*

| Setting | Description |
|---------|-------------|
| **Assistant Name** | Custom name for your AI Co-Pilot (default: "FlyAround AI") |
| **Avatar** | Choose mascot character: Man Pilot, Woman Pilot, and others |

---

## Sound Tab

| Control | Description |
|---------|-------------|
| Volume sliders | Adjust volume per audio category independently |
| Toggle individual sounds | Enable or disable specific announcement types |

Audio categories include cabin announcements, ground crew communication, and system sounds.

---

## Overlay Tab

Configure the **streaming overlay** — a transparent HUD window designed for capture in OBS or Streamlabs:

| Control | Description |
|---------|-------------|
| **Show/Hide Elements** | Toggle which flight data elements are visible |
| **Bot Text** | Customize text for stream chat bots |
| **Live Preview** | See overlay changes in real time |

### Dedicated Overlay Page

Navigate to **Settings → Overlay** for a full-screen live preview and configuration page. Use this page's URL as a **Browser Source** in OBS.
